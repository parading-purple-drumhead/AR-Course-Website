Project 2 - One More Cup of Coffee

Introduction

This project was made as a part of the coursework in CS 428 Virtual, Augmented and Mixed Reality, instructed by Dr. Andrew Johnson at the University of Illinois at Chicago during the Fall 2022 semester. The aim of the project was to create a Virtual Reality scene comprising a commercial kiosk of our choice. I worked on it with my teammate, Advait Pai. Our choice of kiosk for the scene was a convenience store that sells packaged food and stationery.

Demo

How To Use The Application

  1. The scene should load as seen on the right. You may reorient your center at any point using the Oculus button on the right hand controller. You will need to have a 10'x10' clear area to use a application appropriately
  2. Using the Thumbstick on the right hand controller, Navigate to the Welcome mat at the entrance of the store by positioning the green circle and clicking the Thumbstick
  3. On the right, you will see a fridge with multiple food items in the racks. All the food in the fridge can be interacted with by reaching for the food and grabbing it by pressing the Grab Button on either controller. The food can be moved, dropped or tossed in any direction.
  4. To the left of the fridge are the vending machines. The yellow machine on top dispenses Chips, Chocolate and Candy. The desired food can be obtained by reaching out for the associated green button on the machine, which presses it and after a sound cue, dispenses the selected food in the drawer below. The drawer can be opened by grabbing and pulling at the handle, giving access to the food inside which can similarly be taken out.

    Note that only one food item may be dispensed at a time.
  5. Under the yellow machine is the red vending machine that dispenses ice cream bars. The door can be opened via the grab-and-pull mechanism and the ice cream can be accessed inside. Closing the door spawns a new ice cream bar inside.

    Note that closing the door multiple times spawns multiple ice cream bars in a stack
  6. On the left side of the store, there is a stationery rack, which three shelves of stationery items. Similar to the fridge, all the individual items on the stationery rack are interactable via a grab-and-move mechanism.
  7. To the left of the stationery rack is a gumball machine, that dispenses a single gumball upon pressing the button on the machine.

    Note that only one gumball may be dispensed at a time
  8. To the front of the store is the billing desk, manned by a human model of a cashier. The interactable objects on the billing desk are a mouse, a pencil stand and the cashier's name tag.
  9. To the right of the billing desk is a digital kiosk, which upon touching generates a bill on the billing desk after a sound cue.
  10. Both the human models in the scene have voicelines that are triggered by touching the human models.
  11. The seating area present directly opposite the store can be accessed via the teleporter by using the Thumbstick on the right controller

Running the Project

Prerequisites
  • Unity Hub and Unity (Version 2021.3.6f1) - Download Resource
  • Virtual Reality Toolkit (VRTK V4) - Download Resource
  • Oculus Meta Quest 1 or 2 (Optional)
Installing The Project
  1. Access the GitHub link using the following link: GitHub Repository
  2. Click on code and then “Download ZIP”
  3. Open the Zip and extract its contents to a folder with the structure in the image
  4. Open Unity Hub and click on “Open” next to “New Project”. In the File Explorer, navigate to the folder containing the unzipped files. Once found, click “Open”. Make sure your Unity version is 2021.3.6f1.
  5. The project will take some time to load, you may need to wait.
  6. Once the project is loaded, you can close the window shown in the adjacent image
  7. In the Unity Window, in the bottom half, go to the “Project” Tab. In the “Project” explorer, go to “Assets” then “CoffeeShopStarterPack” then “Scenes”. Within the “Scenes” folder, drag the “CoffeeScene” to the project “Hierarchy” as shown in the image. If “Untitled” scene still exists, right click and delete it.
  8. Now we will try to run the project.
  9. To run in the project in the Simulator, in the hierarchy “CameraRigs.SpatialSimulator” must be checked, as shown in the image. Also “CameraRigs.UnityXRPluginFramework” should be unchecked in the hierarchy. Once done, you can press Play to use view the project in Simulator Mode.
  10. You can move the person around using the WASD keys and the mouse. If you want to switch movement between the person and controllers – Press “1” to control the person, Press “2” to control the left hand and Press “3” to control the right hand.
  11. If you want to run the project on the Quest, continue to the next hti.
  12. First, uncheck “CameraRigs.SpatialSimulator” and check “CameraRigs.UnityXRPluginFramework”.
  13. Now in the navigation bar, go to “File” then “Build Settings” and the following window should appear. On the left side select “Android” and the “Switch Platform” below. The switching platform process takes some time.
  14. Note: In case, you’ve already installed Unity without Android support, you can still add Android tools from Unity Hub. On the Installs tab, click the gear icon next to the Unity version to which you want to add the Android tools, and then click Add Modules. Select Android Build Support checkbox, and then select Android SDK & NDK Tools and OpenJDK checkboxes.
  15. Note: When connecting the Quest you may need to give some permission. You should allow all.
  16. Once the switching platform is complete, change the “Texture Compression” to ASTC. In the “Run Device” section, select your Quest headset. Incase it is not visible, press “Refresh”. Otherwise you may need to troubleshoot the detection of the Quest by your device.
  17. Once the Quest is set as “Run Device”, click on “Build and Run”.
  18. This will pop up a new window to save the .apk file. You should save this anywhere outside the project folder and with any name.
  19. Once the .apk is saved, it will be automatically deployed on the Quest. You are now set.

Performance Analysis

Convenience Store
  • The scene when viewed entirely from in front makes the application run at around 60 frames per second (henceforth fps)
  • Upon moving closer and viewing the kiosk from in front, the fps boosts up to about 65-70 fps.
  • Looking to the left of the store, the fps falls to roughly 45-50 fps. This is due to the high polygon count of the the building in the backgrounf combined with the stationery rack and its objects.
  • Viewing the right side of the store makes the fps go back up to 70-75. This is due to the low polygon count of the building in the background
Seating Area
  • Looking at the seating area gives gives a stable framerate of 65-70 fps.
Conclusion

The application on average runs at about 60 fps, never dipping below 45. The reason it dips to 45 is due to the presence of high polygon building models in the background of the left side of the store.

VRTK Spatial Simulator vs Oculus Meta Quest Headset

Unity, along with VRTK 4, allows users to test their Virtual Reality (VR) applications within the Unity Editor itself using the VRTK Spatial Simulator tool. It is a handy tool for those that do not own VR headsets, such as the Oculus Meta Quest or HTC Vive. It allows a user to develop and test changes made to their VR scene using a mouse and keyboard directly in the Player in the Unity Editor. Users can look around the scene by moving around their mouse in the direction they want to look and move around the scene using the W,A,S and D keys on the keyboard. The user can press "1" button to control the user, "2" button to control the left controller and the "3" button to control the right controller. While this is does save the user money if they do not happen to own a VR headset, it is very inconvenient to manually change the position of each controller using a keyboard and mouse. They also do not simulate the interactions with objects in the VR scene accurately. There is also a performance shift to account for. What runs well on the user's PC, might not run as well on a VR headset, depending on the model. The user does not get a proper idea of the scale of objects in the scene either. Thus it is hard to scale and position objects as would be appropriate for a human in the scene. Spatial awareness and estimation is also not possible to emulate accurately using this tool. It greatly increases development and testing time and is not suitable for either. The user might still be able to make do with it for very small applications but anything beyond that is close to impossible to build.

On the other hand, building and testing on a VR headset like the Oculus Meta Quest or the HTC Vive streamlines the development process greatly for the user, when compared to the VRTK Spatial Simulator tool. The user gets an accurate simulation of interactions with objects in the VR scene. The overall process of interaction is seamless, wherein the user can reach for and grab objects with ease as opposed to trying to set the position of the controllers via a keyboard and mouse. These leads to a development experience free of unnecessary frustration caused by using the Spatial Simulator tool. The user can walk around and also turn their head to better understand spatial separation and positioning of objects within the scene. The performance of the application can also be directly gauged in the headset itself, allowing users to optimize their applications better. A slight inconvenience when using a VR headset to test is that the user needs to keep rebuilding or patching changes made in the application to the headset to test those changes. This causes significant annoyance when trying to test for minor changes. In conclusion, it is my opinion that using a dedicated VR headset for the purpose of development is worth the investment.
