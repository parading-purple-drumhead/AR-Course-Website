Project 3 - Our House

Introduction

This project was made as a part of the coursework in CS 428 Virtual, Augmented and Mixed Reality, instructed by Dr. Andrew Johnson at the University of Illinois at Chicago during the Fall 2022 semester. The aim of the project was to create a Virtual Reality scene on a building scale, where we modified the atrium of the upcoming CDRLC building. I worked on it with my teammate, Advait Pai. Our choice of theme for the scene was Christmas and we put various Christmas decorations up in parts of the atrium.

Demo

How To Use The Application

  1. The scene should load as seen on the right. You may reorient your center at any point using the Oculus button on the right hand controller. You can establish a stationary boundary to use the application appropriately
  2. Use the Thumbstick on the right hand controller to navigate around the scene by positioning the green circle and clicking the Thumbstick
  3. Use the Thumbstick on the left hand controller to use the teleporters in the scene by positioning the green circle on one of the teleporters and clicking the Thumbstick
  4. Most small objects in the scene can be interacted with by placing the controller over them and holding down the grab button on the controller, to pick and grab them.
  5. The two levers in the scene can be grabbed and toggled to induce a change in the scene.

    Note that for the change to be reproduced, the lever has to be toggled again to the ON position
  6. The ATM, Hot chocolate station and basketball machine can be operated by pressing the button on them

    Note that an empty paper cup from the scene needs to be placed below the hot chocolate station's nozzle for it to work upon pressing the button
  7. The lighting scheme can be toggled between day and night by pressing the red button between the ATM and the staircase on the first floor
  8. Certain objects including the human models produce a sound upon touching them using the controllers

Running the Project

Prerequisites
  • Unity Hub and Unity (Version 2021.3.6f1) - Download Resource
  • Virtual Reality Toolkit (VRTK V4) - Download Resource
  • Oculus Meta Quest 1 or 2 (Optional)
Installing The Project
  1. Access the GitHub link using the following link: GitHub Repository
  2. Click on code and then “Download ZIP”
  3. Open the Zip and extract its contents to a folder with the structure in the image
  4. Open Unity Hub and click on “Open” next to “New Project”. In the File Explorer, navigate to the folder containing the unzipped files. Once found, click “Open”. Make sure your Unity version is 2021.3.6f1.
  5. The project will take some time to load, you may need to wait.
  6. Once the project is loaded, you can close the window shown in the adjacent image
  7. In the Unity Window, in the bottom half, go to the “Project” Tab. In the “Project” explorer, go to “Assets” then “CoffeeShopStarterPack” then “Scenes”. Within the “Scenes” folder, drag the “CoffeeScene” to the project “Hierarchy” as shown in the image. If “Untitled” scene still exists, right click and delete it.
  8. Now we will try to run the project.
  9. To run in the project in the Simulator, in the hierarchy “CameraRigs.SpatialSimulator” must be checked, as shown in the image. Also “CameraRigs.UnityXRPluginFramework” should be unchecked in the hierarchy. Once done, you can press Play to use view the project in Simulator Mode.
  10. You can move the person around using the WASD keys and the mouse. If you want to switch movement between the person and controllers – Press “1” to control the person, Press “2” to control the left hand and Press “3” to control the right hand.
  11. If you want to run the project on the Quest, continue to the next hti.
  12. First, uncheck “CameraRigs.SpatialSimulator” and check “CameraRigs.UnityXRPluginFramework”.
  13. Now in the navigation bar, go to “File” then “Build Settings” and the following window should appear. On the left side select “Android” and the “Switch Platform” below. The switching platform process takes some time.
  14. Note: In case, you’ve already installed Unity without Android support, you can still add Android tools from Unity Hub. On the Installs tab, click the gear icon next to the Unity version to which you want to add the Android tools, and then click Add Modules. Select Android Build Support checkbox, and then select Android SDK & NDK Tools and OpenJDK checkboxes.
  15. Note: When connecting the Quest you may need to give some permission. You should allow all.
  16. Once the switching platform is complete, change the “Texture Compression” to ASTC. In the “Run Device” section, select your Quest headset. Incase it is not visible, press “Refresh”. Otherwise you may need to troubleshoot the detection of the Quest by your device.
  17. Once the Quest is set as “Run Device”, click on “Build and Run”.
  18. This will pop up a new window to save the .apk file. You should save this anywhere outside the project folder and with any name.
  19. Once the .apk is saved, it will be automatically deployed on the Quest. You are now set.

VR As An Advanced Architectural Tool

Advancements in modern technology are a great reason for traditional activities being a lot more cost-effective now. Architecture is no stranger to the same. There are multiple facets to it, like infrastructure layouts and interior design. While both of those activities are modernized in the form of computer aided design, or simply, CAD, virtual reality systems add a whole new dimension of understanding to the field of architecture. Contextual awareness is the dimension being talked about here. Users get to immerse themselves in a digitally generated scene and experience an environment in first person perspective. This heightens their understanding of 3D positioning and spacing of objects a lot more than they would from a 2D screen. The other major benefit is getting to understand simulated physical spaces without having to actually build anything. This could help with iterative design processes in which feedback from potential clients with every iteration is crucial. Based on present day software and hardware available, the cost of production could be greatly reduced in architectural projects. Scale is not an obstacle either. Anything from small and simple to massive and complex could be built and tested using VR systems.

I would use this application of VR systems in multiple scenarios. The first could be a pitch for a new bit of infrastructure on a campus. People get to experience first hand what being in the proposed environment would be like. Feedback provided could be a lot more meaningful given that the users bear a much deeper understanding of the scene. Another instance of using this technology would be giving students and their parents a virtual tour of the campus without actually needing to be on-site for it. They could explore the entire campus in a first person perspective from the comfort of their homes. It could also be used to show off potential offerings at the school. This could play a crucial part in helping students and their parents pick a school. A similar application of this concept is pitching public infrastructure to politicians and letting them experience it themselves before they sign off on it, thus allowing them to make an informed decision before any major financial resources get invested into it.

The drawback of this application of VR systems is that not everyone owns a VR headset, thus limiting how widely the concept can be applied. In firms dedicated to design and architecture, it is feasible, but in more populated insitutions like universities, it is a lot harder to implement as VR headsets are not very common. However, with the advancement of technology, VR headsets will become a lot more affordable and thus commonplace, which will open a lot more avenues for VR to be applied as discussed.
